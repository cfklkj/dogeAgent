## ✅ 已实现工具

### 1. get_weather - 天气查询工具

**基本信息**:
- **名称**: get_weather
- **描述**: 查询指定城市的实时天气
- **状态**: ✅ 已实现
- **优先级**: ⭐⭐⭐

**功能描述**:
通过风和日丽 API 查询指定城市的实时天气信息，包括温度、天气状况、湿度等。

**参数**:
| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| location | str | 是 | 城市名称，如"北京"、"上海" |

**返回格式**:
```
汪！{城市}的天气：🌡️ {温度} | ☁️ {天气状况} | 💧 湿度{湿度}
```

**使用示例**:

示例 1: 查询北京天气
```
用户："北京天气怎么样？"
调用：get_weather("北京")
回复："汪！北京的天气：🌡️ 22°C | ☁️ 多云 | 💧 湿度 65%"
```

示例 2: 查询上海天气
```
用户："上海今天会下雨吗？"
调用：get_weather("上海")
回复："汪！上海的天气：🌡️ 18°C | ☁️ 小雨 | 💧 湿度 80%...记得带伞哦！"
```

示例 3: 位置不明确
```
用户："天气怎么样？"
回复："汪！我需要知道你在哪个城市呀！🐕 告诉 Doge 你想查哪个城市的天气，比如"北京"、"上海"、"广州"... 这样我才能帮你查天气哦！"
```

**错误处理**:
- 城市不存在 → "汪...这个城市好像不存在呢"
- API 失败 → "汪...天气查询暂时不可用"
- 无位置信息 → "汪！请告诉我你想查哪个城市的天气呀？"

**依赖**:
- 风和日丽 API (https://devapi.qweather.com/v7)
- API Key: QWEATHER_API_KEY

**代码位置**:
- 工具实现：`tools/weather_tool.py`
- 工具注册：`tools/tool_registry.py`

## 🔧 工具开发指南

### 添加工具的步骤

1. **创建工具文件**
   - 在 `tools/` 目录创建 `xxx_tool.py`
   - 实现工具函数

2. **注册工具**
   - 在 `tools/tool_registry.py` 添加工具
   - 使用 `@tool` 装饰器

3. **更新文档**
   - 在 `TOOLS.md` 添加工具信息
   - 更新工具清单表格

4. **测试工具**
   - 编写测试用例
   - 验证工具功能

5. **集成到 Agent**
   - 更新 `agent/factory.py`
   - 确保工具可用

### 工具函数模板

```python
from langchain_core.tools import tool

@tool
def tool_name(param1: str) -> str:
    """
    工具描述
    
    Args:
        param1: 参数描述
    
    Returns:
        返回值描述
    """
    try:
        # 实现逻辑
        result = do_something(param1)
        return result
    except Exception as e:
        return f"工具执行失败：{str(e)}"
```
