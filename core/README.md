## 📚 文件结构

dogeAgent 的核心文件位于 `core/` 目录下，包含以下文件：

```
core/
├── README.md          # 本文档 - 总体说明
├── SOUL.md           # 灵魂定义 - 核心价值观
├── IDENTITY.md       # 身份特征 - 人格和沟通风格
├── MEMORY.md         # 记忆存储 - 用户信息和历史
├── COLLABORATION.md  # 协作协议 - 工具使用规范
└── TOOLS.md          # 工具清单 - 可用工具详情
```

## 🎯 使用方法

### 启动时
1. 读取 `SOUL.md` 确定核心价值观
2. 读取 `IDENTITY.md` 确定人格特征
3. 读取 `MEMORY.md` 加载用户记忆
4. 根据 `COLLABORATION.md` 准备工具
5. 参考 `TOOLS.md` 了解工具详情

### 运行时
- 根据 `SOUL.md` 和 `IDENTITY.md` 指导回复风格
- 使用 `MEMORY.md` 中的信息进行对话
- 按照 `COLLABORATION.md` 调用工具
- 参考 `TOOLS.md` 使用具体工具

### 对话后
- 更新 `MEMORY.md` 记录新信息
- 根据反馈调整行为

### 手动更新
- 修改对应文件

### 自动更新
- `MEMORY.md`: 对话中自动学习 (待实现)
- `TOOLS.md`: 工具变更时更新

- 每个版本有变更记录
- 回滚到历史版本

## 🛠️ 开发指南

### 添加新工具
1. 在 `tools/` 目录实现工具
2. 在 `tool_registry.py` 注册
3. 更新 `TOOLS.md`
4. 更新 `COLLABORATION.md`
5. 测试工具功能

### 修改人格特征
1. 修改 `IDENTITY.md`
2. 确保与 `SOUL.md` 一致
3. 测试行为变化
4. 记录变更原因

### 更新记忆
1. 从对话中提取信息
2. 更新 `MEMORY.md`
3. 确保信息准确
4. 保护用户隐私

## 📈 与其他模块的关系

```
core/           # 核心定义
  ↓
agent/          # Python 实现
  ↓
tools/          # 工具实现
  ↓
electron/       # 前端展示
```

- **core/**: 定义"是什么"
- **agent/**: 实现"怎么做"
- **tools/**: 提供"能力"
- **electron/**: 展示"界面"
