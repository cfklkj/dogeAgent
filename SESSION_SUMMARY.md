## 🧠 关键决策

### 1. 架构决策
- **工具调用模式**: 使用 LangChain 的 `bind_tools` + 手动执行
- **城市识别**: 预定义城市列表 + 正则匹配"XX 的天气"
- **降级策略**: API 失败时返回模拟数据，保证用户体验

### 2. 核心文件设计
- **人格定义**: 柴犬宠物（Doge），ENFP 性格，友好活泼
- **沟通风格**: 使用 emoji 和语气词（汪！呀！呢！）
- **记忆系统**: 动态更新，记录用户偏好和对话历史
- **工具规范**: 明确使用原则、错误处理流程

### 3. 技术选型
- **天气 API**: 风和日丽 (QWeather) - 免费、准确
- **核心文件**: Markdown 格式，易读易维护
- **加载机制**: 惰性加载，首次使用时初始化

### 工具调用流程
```
1. 用户消息 → LLM
2. LLM 识别意图 → 返回 tool_calls
3. 检测 tool_calls → 提取参数
4. 参数不足 → 调用 _extract_city()
5. 执行工具 → 获取结果
6. 结果反馈给 LLM → 生成最终回复
```
